# Ghost: Thai-to-Russian Transcription & Translation Suite

Ghost — это инструмент для захвата тайской речи и превращения её в структурированные данные с последующим ручным переводом на русский язык. Проект оптимизирован для работы на macOS (Apple Silicon) и обеспечивает полный цикл: от записи звука до формирования готового транскрипта.

## Что реализовано в проекте

### 1. Система захвата аудио (Backend & Client)
*   **Web-интерфейс записи**: Позволяет записывать звук прямо из браузера. Включает визуализатор громкости (VU-meter), спектрограмму, настройки усиления (Gain) и шумоподавления (Gate).
*   **WebSocket сервер**: Реализован на Rust (Axum). Принимает аудиопотоки (WebM/Opus из браузера или сырой PCM из консоли), автоматически нарезает их на сегменты по 2 минуты и сохраняет на диск.
*   **Автоматический TLS**: Файл `build.rs` генерирует самоподписанные сертификаты при сборке, что необходимо для доступа к микрофону в современных браузерах по HTTPS на `localhost` и в локальной сети.
*   **Console Recorder**: Отдельный бинарный файл для записи аудио через системный микрофон без использования браузера.

### 2. Транскрибация (AI Layer)
*   **MLX-Whisper Integration**: Скрипт `transcribe_thai_mac.py` использует модель `whisper-large-v3-mlx`, оптимизированную для GPU Apple Silicon (M1/M2/M3/M4). 
*   **Тайский язык**: Система настроена на распознавание тайского языка с высокой точностью.
*   **Watch-режим**: Скрипт умеет следить за папкой `recordings/` и автоматически транскрибировать новые файлы по мере их появления.

### 3. Обработка данных и БД
*   **SQLite + SQLx**: Все данные (сессии, сегменты, реплики) хранятся в структурированной базе данных `transcripts.db`.
*   **Ingest Tool**: Утилита для синхронизации файлов с диска в базу данных. Она парсит `.txt` файлы от Whisper, извлекает таймкоды и текст, создавая структуру сессий.
*   **Слой сервисов**: Четкое разделение ответственности между Repository (SQL), Service (бизнес-логика) и Handler (API).

### 4. Интерфейс переводчика (UI)
*   **SPA на Vanilla JS**: Легкий и быстрый интерфейс для просмотра транскриптов.
*   **Редактор перевода**: Оператор видит тайский текст с привязкой ко времени и может вводить русский перевод.
*   **Auto-save**: Перевод сохраняется в базу данных в реальном времени (debounce PATCH запросы).
*   **Экспорт**: Функция копирования всей сессии (тайский + русский текст) одним кликом.

## Архитектура

*   **Rust (Backend)**: Axum, Tokio, SQLx, Rustls.
*   **Python (ML)**: MLX, Whisper (large-v3), FFMPEG.
*   **Database**: SQLite (WAL mode).

## Быстрый старт

### Предварительные требования
1. Rust (последняя стабильная версия).
2. Python 3.12+ (рекомендуется `uv`).
3. FFMPEG (для обработки аудио).

### 1. Сборка и запуск сервера
```bash
cargo build
cargo run --bin ghost
```
Сервер будет доступен по адресу `https://localhost:3005`.

### 2. Настройка транскрибатора (macOS)
```bash
uv sync --extra mac
python transcribe_thai_mac.py recordings/ --watch
```

### 3. Синхронизация данных в БД
Для того чтобы новые транскрипты появились в интерфейсе перевода:
```bash
cargo run --bin ingest -- --watch
```

## Структура папок
*   `/src` — исходный код Rust (модели, сервисы, API).
*   `/src/bin` — вспомогательные утилиты (recorder, ingest).
*   `/recordings` — папка для хранения аудио (`.webm`, `.wav`) и текстовых транскриптов (`.txt`).
*   `build.rs` — генерация TLS-сертификатов.
*   `transcribe_thai_mac.py` — основной ML-скрипт.

## API Endpoints
*   `GET /api/sessions` — список всех сессий записи.
*   `GET /api/sessions/:id` — детальная информация о сессии со всеми репликами.
*   `PATCH /api/utterances/:id/translation` — обновление русского перевода.
*   `DELETE /api/sessions/:id` — удаление сессии.
